# 60〜80点のLPモックアップ生成プロンプト（1回で出力）

以下をそのまま貼り付けて使う。

```
あなたは実務レベルのWeb/UIデザイナー兼フロントエンド担当。宿のLP（1ページ）モックアップを**1回の出力で60〜80点**の完成度で作成してください。目的は、最終デザインに近い構成・見た目・コピーのたたき台を短時間で得ること。

# 絶対条件（厳守）
- 1ページLP
- **外部リンクは下記URLのみ**（他OTA・予約サイト・SNS等は禁止）
  - https://www.airbnb.jp/rooms/1557698987387425826
- **表示テキストに「Airbnb」という単語を一切入れない**（タイトル/見出し/注記/FAQ/フッター/ボタン文言/本文すべて）
- CTA文言は「予約する」「料金を見る」「問い合わせる」など一般的な表現にする
- 不確実な情報は断定しない（「詳細は予約ページでご確認ください」程度に留める）
- 誇張表現は避ける

# 必須セクション/要素
- ヒーロー（宿の魅力コピー、CTA、代表写真）
- 予約ボタン（上記URLへリンク）
- 料金の目安（不確実なら“例”として記載し、注記）
- アクセス情報（地名・交通手段は不確実ならプレースホルダ）
- 写真ギャラリー（複数枚）
- 滞在の特徴（静けさ、建築、自然、設備など）
- FAQ（3〜5件）
- 連絡/問い合わせ（予約ページのメッセージ案内）
- フッター（注意事項/規約の簡易記載）

# デザイン参照（万平ホテルサイトの設計思想を抽出）
- **大きなビジュアルと短いコピー**で世界観を作る
- **英語の小さなラベル + 日本語の見出し**の二層タイポ
- ナビに**カテゴリ群 + Menu + 言語切替風**の要素を配置（簡易でOK）
- ヒーローに「短いキャッチ」「サブコピー」「スクロール誘導」の要素を配置（“Scroll / Skip” のようなマイクロUI）
- ヒーロー内に**“In [Place], since [Year]”のような歴史ライン**を1行入れる
- 導入（Introduction）セクションを設け、**2〜4段落の物語的なテキスト**で価値を語る（最後に短い“哲学”フレーズ）
- セクションは**大画像＋テキスト**の組み合わせでリズムを作る
- 「最新情報」風の**小さなリスト**（日付つきの短い項目）を1ブロック入れる
- 下部に**予約ブロック（簡易フォーム風）**を置き、CTAへ誘導
- フッターは**情報量多め・整然**（コンテンツ一覧/案内/所在地/電話の枠組み）

# 画面設計（今回の追加条件）
- **余白を削り、フルブリードの写真を多用**（画面いっぱいのビジュアルが主役）
- ヒーローは**全面写真**（100vh推奨）＋半透明のテキストパネル
- **トップの“被せ画像”は使わない**（ヒーローに重なる小さな画像を置かない）
- ギャラリー/予約ブロックも**横幅いっぱい**に展開する

# デザイン/品質の目標
- **ワイヤーではなく“見た目が成立している”状態**（60〜80点の完成度）
- 余白・タイポグラフィ・色の階調が整っている
- セクションごとに**見出し＋短いリード文**を用意
- カード/チップ/バッジ等のUI要素を使い、単調さを避ける
- 背景に**微細なテクスチャやグラデーション**を入れて奥行きを出す

# トーン/世界観
- 木・土・生成り・深緑を基調に、落ち着いた温かみ
- 伝統的な木造建築と自然の静けさ
- 日本語は上品で読みやすく、英字は控えめ

# 出力形式
- **HTML + CSS + JS を1ファイルで出力**（`<style>` と `<script>` を使用）
- セクション順・見出し構造（h1→h2→h3）を守る
- 画像は仮でOKだが**alt文は具体的に**
- JSは軽い演出のみ（例: セクションのフェードイン、FAQアコーディオン）
- **外部ライブラリなし**

# 期待するアウトプット
- そのまま関係者に共有できる品質
- 具体的なコピーとUI要素が揃っている

# 最終出力
- **HTMLコードのみ**を返す（説明文は不要）
```
